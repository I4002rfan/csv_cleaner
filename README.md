# Bodhitree Test Case Intelligence System

A Django REST Framework backend that enables bulk test case uploads via CSV and AI-powered test case/edge case generation with syllabus-aware context. Built as a POC for integration into the Bodhitree LMS platform at IIT Bombay.

## Project Overview

This system solves three problems for instructors on the Bodhitree platform:

1. **Bulk test case upload** — Instead of entering test cases one by one, instructors upload a CSV file containing all test cases at once.
2. **AI-generated test cases** — Given a problem statement, the AI generates a complete test suite automatically.
3. **AI-generated edge cases** — The AI specifically generates boundary and edge cases that expose common bugs.

All AI generation is **syllabus-aware** — the system knows which topics have been covered in the course so far, and generates test cases appropriate to that context. It also adapts to the exam type (practice, midterm, endterm) and difficulty level (beginner, intermediate, advanced).

## Tech Stack

- **Backend:** Django 6.0.5 + Django REST Framework 3.17.1
- **AI:** Groq API (model: `llama-3.3-70b-versatile`)
- **Database:** SQLite (development) — replace with PostgreSQL for production

## Setup and Installation

**1. Clone the repository:**
```bash
git clone <repo-url>
cd csvCleaner
```

**2. Create and activate virtual environment:**
```bash
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Mac/Linux
```

**3. Install dependencies:**
```bash
pip install -r requirements.txt
```

**4. Set up environment variables:**

Create a `.env` file at the root level:
```
GROQ_API_KEY=your_groq_api_key_here
```

**5. Run migrations:**
```bash
python manage.py makemigrations
python manage.py migrate
```

**6. Start the server:**
```bash
python manage.py runserver
```

Server runs at `http://127.0.0.1:8000/`

## Database Models

### Course
Stores course information and the full parsed syllabus.

| Field | Type | Description |
|-------|------|-------------|
| `id` | Auto | Primary key |
| `course_name` | CharField | Name of the course e.g. "Basic C++" |
| `course_level` | CharField | beginner / intermediate / advanced |
| `syllabus` | JSONField | Parsed hierarchical syllabus (auto-generated by AI) |
| `uploaded_at` | DateTimeField | Timestamp of upload |

### Module
Stores individual modules belonging to a course.

| Field | Type | Description |
|-------|------|-------------|
| `id` | Auto | Primary key |
| `course` | ForeignKey | Links to Course |
| `module_name` | CharField | Name of the module e.g. "Arrays" |
| `order` | IntegerField | Position in syllabus sequence |
| `created_at` | DateTimeField | Timestamp |

### Problem
Stores lab/practice problems with topic tags.

| Field | Type | Description |
|-------|------|-------------|
| `id` | Auto | Primary key |
| `course` | ForeignKey | Links to Course |
| `title` | CharField | Problem title |
| `description` | TextField | Full problem statement |
| `module_tags` | JSONField | List of module names this problem tests e.g. `["Loops", "Functions"]` |
| `subtopic_tags` | JSONField | Optional — more specific topic tags |
| `lecture_tags` | JSONField | Optional — lecture-level tags |
| `problem_level` | CharField | beginner / intermediate / advanced |
| `exam_type` | CharField | practice / midterm / endterm |

## API Endpoints

Base URL: `http://127.0.0.1:8000/csvUploder/`

---

### 1. Upload CSV Test Cases
**`POST /csvUploder/upload/`**

Accepts a CSV file containing test cases, validates each row, and returns structured data.

**Request:** `multipart/form-data`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `file` | File | Yes | CSV file with test cases |

**CSV Format:**
```
input,expected_output,marks,type
5,120,2,normal
0,1,1,edge
-1,error,1,corner
```

**Validation Rules:**
- File must be `.csv`
- Required columns: `input`, `expected_output`, `marks`, `type`
- `marks` must be an integer (strings like `"5"` are auto-converted)
- `type` must be one of: `normal`, `edge`, `corner`
- Missing fields in a row → row rejected with reason
- Invalid values → row rejected with reason

**Success Response:**
```json
{
    "success_count": 2,
    "success_rows": [
        {
            "input": "5",
            "expected_output": "120",
            "marks": 2,
            "type": "normal"
        }
    ],
    "error_count": 1,
    "errors": [
        {
            "row": 4,
            "error": "Marks must be an integer"
        }
    ]
}
```

---

### 2. Upload Course Syllabus
**`POST /csvUploder/upload-syllabus/`**

Accepts raw syllabus text, uses AI to parse it into a structured hierarchy, and saves it to the database.

**Request:** `application/json`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `course_name` | String | Yes | Name of the course |
| `course_level` | String | Yes | beginner / intermediate / advanced |
| `syllabus` | String | Yes | Raw syllabus text in any format |

**Example Request:**
```json
{
    "course_name": "Basic C++",
    "course_level": "beginner",
    "syllabus": "M) Variables\nData Types\nM) Loops\nWhile loop\nFor loop\nM) Functions\nBasic functions\nRecursion"
}
```

**Success Response:**
```json
{
    "message": "Syllabus uploaded successfully",
    "course_id": 1,
    "course_name": "Basic C++",
    "syllabus": {
        "modules": [
            {
                "order": 1,
                "name": "Variables",
                "subtopics": [...]
            }
        ]
    }
}
```

**Notes:**
- The `course_id` returned here is used in test case generation endpoints
- Syllabus can be in any format — numbered, bulleted, indented, `M)` prefixed etc.
- AI handles parsing automatically

---

### 3. Generate Test Cases
**`POST /csvUploder/generate-testcases/`**

Generates a complete test suite for a problem using AI, with optional syllabus-aware context.

**Request:** `application/json`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `problem_statement` | String | Yes | Description of the problem |
| `input_format` | String | Yes | How input is structured |
| `output_format` | String | Yes | Expected output format |
| `constraints` | String | Yes | Input constraints e.g. `"1 <= n <= 1000"` |
| `course_id` | Integer | No | ID of the course (enables syllabus context) |
| `module_tags` | Array | No | Modules this problem tests e.g. `["Loops", "Functions"]` |
| `exam_type` | String | No | practice / midterm / endterm (default: practice) |
| `problem_level` | String | No | beginner / intermediate / advanced (default: beginner) |

**Example Request:**
```json
{
    "problem_statement": "Find GCD of two numbers",
    "input_format": "two integers a and b on one line",
    "output_format": "single integer",
    "constraints": "1 <= a, b <= 1000",
    "course_id": 1,
    "module_tags": ["Loops", "Functions"],
    "exam_type": "midterm",
    "problem_level": "intermediate"
}
```

**Success Response:**
```json
{
    "test_cases": [
        {
            "id": 1,
            "input": "12 15",
            "expected_output": "3",
            "constraint_tested": "typical values",
            "marks": 5,
            "type": "normal",
            "purpose": "Simple GCD calculation",
            "notes": "Basic correctness check"
        }
    ],
    "metadata": {
        "coverage": ["boundary", "corner", "stress"],
        "total_test_cases": 10,
        "total_marks": 100
    }
}
```

**How difficulty affects output:**

| exam_type | problem_level | What AI generates |
|-----------|--------------|-------------------|
| practice | beginner | Normal + simple edge cases only |
| practice | intermediate | Normal + edge + simple corner cases |
| practice | advanced | Normal + edge + corner + moderate stress |
| midterm | beginner | Normal + edge + basic corner cases |
| midterm | intermediate | Normal + edge + corner cases |
| midterm | advanced | Normal + edge + corner + moderate stress |
| endterm | beginner | Normal + edge + corner + small stress |
| endterm | intermediate | All types including stress |
| endterm | advanced | All types + adversarial + full complexity checks |

### 4. Generate Edge Cases
**`POST /csvUploder/generate-edgecases/`**

Generates boundary and edge cases specifically designed to expose common bugs.

**Request:** Same fields as `generate-testcases`

**Success Response:**
```json
{
    "edge_cases": [
        {
            "id": 1,
            "input": "1 1",
            "expected_output": "1",
            "edge_type": "min_value",
            "purpose": "Minimum valid input",
            "notes": "Tests handling of smallest possible values"
        }
    ],
    "metadata": {
        "total_edge_cases": 6,
        "coverage": ["min_value", "max_value", "overflow_risk", "equal_numbers"]
    }
}
```

## Utility Functions

Located in `csvUploder/utils.py`

---

### `parse_ai_json(raw_text)`
Cleans and parses raw AI response text into a Python dictionary.

Handles cases where the AI wraps JSON in markdown code fences like:
````
```json
{...}
```
````

**Parameters:**
- `raw_text` (str) — Raw response string from Groq API

**Returns:** Python dictionary

**Raises:** `json.JSONDecodeError` if parsing fails

---

### `get_coverage(course_id, module_tags)`
Calculates which syllabus topics are covered and not yet covered based on a problem's module tags.

**How it works:**
1. Fetches the course syllabus from DB using `course_id`
2. Extracts ordered module names from the syllabus JSON
3. Finds the latest position of any tag in `module_tags`
4. Everything up to that position = covered
5. Everything after = not yet covered

**Parameters:**
- `course_id` (int) — ID of the course in DB
- `module_tags` (list) — List of module names e.g. `["Loops", "Functions"]`

**Returns:**
```python
{
    'topics_covered': ['Variables', 'Conditionals', 'Loops', 'Functions'],
    'topics_not_covered': ['Arrays', 'Structures'],
    'focus_topics': ['Loops', 'Functions']
}
```

**Returns `None` if:**
- Course not found
- Syllabus is empty
- No matching module tag found

---

### `get_difficulty_context(exam_type, problem_level)`
Returns a detailed difficulty profile string to inject into the AI prompt.

**Parameters:**
- `exam_type` (str) — practice / midterm / endterm
- `problem_level` (str) — beginner / intermediate / advanced

**Returns:** String with detailed instructions for the AI about test case complexity, coverage expectations, and what to avoid.

**All 9 combinations are explicitly handled.** Falls back to a standard profile for unknown combinations.

## AI Prompts

Located in `csvUploder/prompts/`

### `testcase_prompt.txt`
Instructs the AI to generate a complete test suite covering normal, edge, corner, and stress cases. Returns strict JSON only.

### `edgecase_prompt.txt`
Instructs the AI to generate only boundary and edge cases with `edge_type` labels and coverage metadata.

### `syllabus_prompt.txt`
Instructs the AI to parse raw syllabus text of any format into a structured JSON hierarchy of modules, subtopics, and lectures.

**All prompts follow these rules:**
- Return only valid JSON
- No markdown, no code fences, no explanations
- Validated before output

## Integration Guide

When integrating into the main Bodhitree repository:

**Step 1 — Copy these files directly:**
- `csvUploder/prompts/` — entire folder
- `csvUploder/utils.py` — all three utility functions

**Step 2 — Check existing models:**
- If Bodhitree already has `Course` and `Problem` models, map fields accordingly
- The key fields needed: `Course.syllabus` (JSONField), `Problem.module_tags` (JSONField)

**Step 3 — Replace Groq with BodhiBot:**
- In `views.py`, find the `client.chat.completions.create()` calls
- Replace with BodhiBot API call using the same message structure
- Response parsing via `parse_ai_json()` stays the same

**Step 4 — Replace mock DB save with real models:**
- In `upload_csv`, replace `return Response(success_rows)` with actual `TestCase.objects.create()` calls
- Use Bodhitree's existing TestCase model schema

**Step 5 — Add to existing URLs:**
- Add the four endpoints to Bodhitree's existing `urls.py`
- Follow their URL naming convention

**Step 6 — Environment:**
- Add `GROQ_API_KEY` (or BodhiBot credentials) to their `.env`
- Add `corsheaders` to their `INSTALLED_APPS` if not already present
